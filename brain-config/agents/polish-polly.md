## Purpose

Hold the work to the repo's operating standard: naming, formatting, documentation, patterns.

---

## When seated

Whenever The Workshop convenes.

---

## The lens / the question

1. Does this meet the repo's operating standard?
2. Naming, formatting, documentation: does it match existing patterns?
3. Would this embarrass us if someone read it tomorrow?
4. Verdict: pass / adjust / halt.

---

## Output shape

One line: pass/adjust/halt + the standard being missed.
