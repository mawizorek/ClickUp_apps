About Inbox Digest

A data-rendered audit surface for the Gmail inbox sweep. The sweep reads the inbox, merges keeper content into ClickUp tasks, and this app renders what was captured so the whole loop is auditable.

What it is (and isn't)

This is not an email viewer. It renders metadata only: who, what, when, thread size, read state, attachment names, and a two-line snippet, never full body text. The current landing page is the Field Capture Matrix: proof of exactly which fields the sweep can pull.

How it will work

The matrix maps 1:1 to a rolling data/inbox-state.json that Brain rewrites each sweep. The file is the audit trail: if a session ends mid-sweep, another agent reads it and continues. The dashboard view (state + action buckets) is the next build; index.html repoints the landing to it with one constant when it lands.

Built on the gold-standard template: shared chrome, dual drawers, and the shared/themes spine. Open Settings and switch themes to reskin the whole app live.

Hard limits

The sweep cannot read Gmail labels/folders, mark messages read, archive, or send. Every state change is a manual click; this app only shows what was captured and what to do next. Attachments travel only when you forward an email into ClickUp.
