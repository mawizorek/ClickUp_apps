A theme is a color and a feeling.

Three entities: a Color (hue only, the 17 tokens), a Feeling (form: type, corners, line weight, gradient, shadow), and a Theme — the named pairing of exactly one of each. Apps are skinned with a Theme. Every object below is a copy-me spec card: the live shape plus its exact FileMaker recipe. Pick a blessed theme, or mix any color × feeling to explore.

Buttons — full set with states

Fill + corner + one shadow + text. Primary carries a 2-stop gradient baked from the active accent; the pressed state swaps the outer shadow for an inner shadow (the infill rounded-rect look).

Shells, toolbar & nav

Depth = surface gradient + border + one outer shadow. One shadow per object, exactly FileMaker's budget.

Fields — editable, focus, readonly

Focus = inner shadow + accent border. Readonly drops to a surface fill.

Pills, badges & callout

Pill radius is a feeling decision (near-square in Sharp Racing). Semantic fills come from good/warn/bad/info.

Table & portal rows

Header, standard, alternating, selected. Selected = accent-soft; alt = surface-2.

Color swatches — 17 tokens for McLaren

The active color's full token set with roles. The palette half of "pick one and pick one": the only colors any object may use.

To build this natively: for each object card, create one FileMaker Style using its recipe + these swatch colors, then save all Styles into one FileMaker Theme. Toolbox + limits: FILEMAKER-CAPABILITIES.md · structure: THEME-SYSTEM.md.
