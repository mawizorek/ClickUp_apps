Theme Render Board · reference build: sharp-mclaren

A theme is a color and a feeling.

Three entities: a Color (hue, 17 hex tokens + a gradient-deep stop), a Feeling (form: type, corners, line weight, shadow, gradient angle), and a Theme — the named pairing of exactly one of each. Values come straight from the TSV grids (colors.tsv, feelings.tsv); every color is a literal hex, and the gradient is two explicit hex stops. Nothing is computed at runtime, so what you see is what you copy into FileMaker.

Buttons — full set with states

Fill + corner + one shadow + text. Primary uses the two-stop hex gradient (accent → accent-deep); pressed swaps the outer shadow for an inner shadow (the infill rounded-rect look).

Shells, toolbar & nav

Depth = surface fill + border + one outer shadow. One shadow per object, FileMaker's budget.

Fields — editable, focus, readonly

Focus = inner shadow + accent border. Readonly drops to a surface fill.

Pills, badges & callout

Pill radius is a feeling decision. Semantic fills come from good/warn/bad/info.

Table & portal rows

Color swatches — hex tokens for McLaren

The active color's full token set with hex values + roles. Copy these straight into FileMaker. The two gradient stops (accent, accent-deep) are both here as literal hex.
