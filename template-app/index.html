App Build Standard

This folder is the scaffold every new app starts from. Copy it, rename it, build on it. Everything here defines the shape, docs, and workflow that all apps in this repo follow.

Required folder shape

your-app/
├── index.html ← the app itself + Pages entry point
├── README.md ← permanent docs, launch link, architecture
├── next-build-spec.md ← current build contract (overwritten each cycle)
├── data.json ← optional, for data-separated apps
└── source/ ← chunk set for Brain readback (if app > 30KB)

README structure (every app)

  • Title + launch button at the very top
  • Status line (Live / Source of truth)
  • What it does
  • How to use it
  • Infrastructure table (files + roles + update frequency)
  • Architecture (non-obvious technical decisions)
  • Version history (highlights, commit history is authoritative)
  • Related links (ClickUp task, revision history)
  • Roadmap

Build workflow

  • Brainstorm Gate — 7 teams, all fired before any spec is written
  • Session 1 (Spec Agent) — hash out changes, commit next-build-spec.md
  • Session 2 (Render Agent) — reads spec, applies diffs, delivers artifact
  • Pre-ship gates — Syntax Gate + Runtime/Sandbox Gate, both mandatory
  • Ship — version comment, twin capture, field updates
