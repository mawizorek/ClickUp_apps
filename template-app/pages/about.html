About this template

A second page, proving the router. The shell (index.html) never stores a page; it loads these partials from pages/ by hash route and repoints the landing with a single DEFAULT_PAGE constant.

How it fits together

The header, menu, settings drawer, and footer are built once by chrome.js and persist across every page. Switching pages swaps only the content in #view, so the chrome and the active theme never flicker.

The theme spine

All styling references shared/themes through var(--token). Pick a theme in Settings and it applies live via resolve.js and persists to this browser. A static themes.css paints the correct theme on first load so there is no unstyled flash.

Need a color the 17 tokens can't express? Don't inline it. Flag a theme expansion (see README) so the spine stays the single source of color.

Copy me

This folder is the starting point for every new app. Copy it, rename it, swap the APP_* constants at the top of index.html, add your pages, pick a theme. See CONFORMANCE.md for the checklist an audit compares against.
